МАСТЕР-ТЗ — «WB Radar & China Matcher» (десктоп)
Одно десктопное приложение, две вкладки:

Матчер China (головной пайплайн) — кидаю товар WB (артикул / ссылка / фото) → нахожу его 1:1 на китайских площадках → забираю полочное видео.
Разведка WB — нахожу вирусные товары, парсю карточки и отзывы, тяну видео из отзывов, вытаскиваю боли/желания/страхи, пишу описания и хуки.

Методика: Harness Engineering (5 подсистем). Одна фича = один промпт. Завязки + тесты + критерий «готово» у каждой. Без заглушек. На выходе — собранный .exe, готовый к бою.

Стек: Python 3.11, Flet (десктоп), Playwright, open_clip + imagehash, httpx, pydantic, pytest. LLM-слой: OpenRouter / Z.AI(GLM) / Groq / Ollama(локально) + опц. ChatGPT-web.


ЧАСТЬ I. Как этим пользоваться (для тебя)
Порядок скармливания агенту
Отдаёшь агенту AGENTS.md (Часть III) + feature_list.json (Часть IV) + progress.md (Часть V). Это его харнесс.
Стартовый промпт (Часть II) — агент подтверждает архитектуру, код не пишет.
Дальше фичи строго по одной из Части VI. После каждой агент: пишет код без заглушек → гоняет тесты → обновляет progress.md и статус в feature_list.json → ждёт твоего «ок».
Ты проверяешь по блоку «Готово когда» и «Что смотреть мне» у фичи. Не сошлось — возвращаешь на доработку, вперёд не пускаешь.
Логика использования готового приложения
Сценарий A (быстро): вкладка Матчер China → артикул WB → результаты 1:1 + кнопки «видео/скачать».
Сценарий B (полный): вкладка Разведка WB → ниша → топ вирусных → выбираешь товар → VoC + хуки → кнопка «В Матчер» перекидывает артикул в первую вкладку за полочным видео. Видео из отзывов WB тянется тут же.

# AGENTS.md — WB Radar & China Matcher

## Старт сессии

1. Прочитай progress.md → найди активную фичу (status=in_progress) или первую todo с готовыми завязками.

2. Прочитай её спецификацию в TZ.md (Часть VI).

5. Обнови progress.md (что сделал, какие тесты прошли, следующий шаг) и status в feature_list.json.

6. Заполни session-handoff.md и остановись. Жди подтверждения пользователя.

## Что строим

Десктоп-приложение, 2 вкладки:

- Матчер China (приоритет): товар WB (артикул/ссылка/фото) -> 1:1 на 1688/Alibaba/Taobao -> видео.

- Разведка WB: вирусные товары -> карточки+отзывы -> видео из отзывов -> боли/желания/страхи -> хуки/описания.

## Архитектура (инварианты — не нарушать)

- core/ — общий фундамент: config, wb_public (публичные эндпоинты WB), browser (Playwright),

  llm (провайдеры), storage (sqlite+файлы), models (pydantic). Дублировать WB/LLM/браузер в модулях ЗАПРЕЩЕНО.

- matcher/ — китайский поиск по фото, CLIP/pHash ранкер, извлечение видео карточки.

- harvest/ — детектор вирусных, сбор отзывов, VoC-анализ, хуки, видео из отзывов, описания.

- gui/ — Flet, вкладки. Первая вкладка по умолчанию — Матчер China.

## Правила

- Одна активная фича. Каждая = отдельный коммит вида "F07: llm provider layer".

- БЕЗ ЗАГЛУШЕК: никаких pass/TODO/"реализуем позже". Фича либо сделана полностью, либо не закрыта.

- Тесты обязательны. Перед "готово" — pytest зелёный (без маркера live).

- LLM-провайдер выбирается из config, код не завязан на конкретного вендора.

## Безопасность и границы (НЕ нарушать)

- НЕ обходим капчи/антибот/WAF. Капча -> ручной ввод пользователя в видимом окне ИЛИ skip площадки с логом.

- Публичные эндпоинты WB (card.wb.ru и т.п.) — неофициальны и против п.9.9.6 оферты WB. Используем с

  умеренным rate-limit, ретраями и паузами. Это решение пользователя (ToS, не закон).

- Чужой авторский видеоконтент (отзывы) не перезаливаем 1:1 — храним как референс/материал. Описание пишем своё.

- Секреты (API-ключи, сессии) — только в .env / sessions/, в git не коммитим.

## Definition of Done (для любой фичи)

[ ] код без заглушек  [ ] юнит-тесты на фикстурах зелёные  [ ] критерий "Готово когда" выполнен

[ ] progress.md и feature_list.json обновлены  [ ] не сломаны прошлые фичи (полный pytest зелёный)

## Стоп-правила

- Фича требует обхода защиты -> стоп, сообщить, предложить ручной/skip режим.

- Эндпоинт WB вернул 400/403 -> пересобрать заголовки из живой сессии, ретрай, затем skip с логом. Не выдумывать данные.

ЧАСТЬ VIII. Тесты — политика (без заглушек)
Юнит на каждый модуль, данные — сохранённые реальные фикстуры (fixtures/*.json, *.html). Это не заглушка функционала, а зафиксированный ответ сети для детерминизма.
Контракт-тесты LLM-провайдеров: единый интерфейс, мок транспорта.
Live-смоук под @pytest.mark.live — реальные WB/China/LLM, запускаются вручную (pytest -m live), не в обычном прогоне.
E2E на фикстурах — весь путь вход→выход.
Правило: фича не закрыта, пока pytest -m "not live" не зелёный и критерий «Готово когда» не выполнен. Никаких pass/TODO/«доделаю позже» в коде.

ЧАСТЬ X. Это десктоп — почему так
Flet даёт нативное десктоп-окно из Python и собирается в .exe одной командой (flet pack) — под твой стек идеально, без отдельного фронта.
Одно приложение, две вкладки, общий core/ — никакого дублирования WB-клиента/LLM/браузера между матчером и разведкой.
LLM подключается через конфиг: дефолт — OpenRouter (один ключ = и GPT openai/*, и Claude, и др.), плюс Z.AI(GLM), Groq, локальный Ollama для приватных/оффлайн-прогонов, и ChatGPT-web как запасной вариант через личный аккаунт (нестабильно, против ToS — на твой риск).
Приоритет интерфейса: вкладка Матчер China открывается первой, как ты и просил; «Разведка WB» питает её артикулами через кнопку «В Матчер».
